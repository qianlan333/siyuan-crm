## 引用核验

- Required candidate grep shows PR-9 deleted modules are no longer present in `wecom_ability_service/http/__init__.py` as registered modules or registrars.
- `aicrm_next/main.py` does not import `production_compat_router`、`production_compat_wildcard_router`、`forward_to_legacy_flask`、`legacy_flask_facade` or `X-AICRM-Compatibility-Facade` response path.
- Current route resolution counters remain zero for production compatibility, legacy fallback, unknown owner, undocumented routes, and deleted-but-registered routes.

## PR-9 test migration

1. `tests/test_automation_conversion_v1.py` was removed because its remaining value was tied to legacy Flask monkeypatch seams; active behavior is covered by the existing Next timer, member action, workspace runtime, route-precedence, no-real-side-effect, and production route resolution tests.
2. `tests/test_customer_automation_webhook_registry_lifecycle.py` was removed because route registry lifecycle assertions are now owned by Next route registry / production route resolution tests; activation and retry side-effect boundaries remain covered by customer webhook Next tests.
3. `tests/test_pr9_legacy_prune.py` locks the deleted file set, verifies no runtime import remains, and checks this inventory matches filesystem state.
