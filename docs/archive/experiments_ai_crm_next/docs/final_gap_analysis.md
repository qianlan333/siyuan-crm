# AI-CRM Next Final Gap Analysis

This document summarizes the gap between the current AI-CRM Next experiment and a production replacement for the old Flask AI-CRM service. The current system is strong in API contracts, fixture parity, bounded-context structure, and selected PostgreSQL-ready repository layers. It is not production-ready because real production data, real external adapters, live dual-run evidence, and cutover controls are still missing.

## Highest Priority Gaps

1. Real PostgreSQL integration tests have not been proven in this environment.
2. Production data migration/backfill is not designed.
3. Real WeCom, WeChat OAuth, WeChat Pay, Alipay, OpenClaw, cloud storage, and WeCom media adapters are not connected.
4. Most frontend pages are partial adapters, not route-by-route production parity.
5. Automation workflow/runtime is first-slice only.
6. Observability, audit, idempotency, rollback, and route-level smoke are not production hardened.
