# AI-CRM Next Module Status Matrix

Historical note: this experiment document records pre-retirement readiness and canary evidence. Current main has D1-D3 readonly owners retired/deleted, D4-D6 readonly owners retired/tombstoned, and D6.5 no-reference cleanup completed. The production signoff and canary status values below are retained as historical experiment evidence only; they do not reopen old Flask readonly route ownership or authorize D7 write/external/runtime deletion.

Status vocabulary:

- `not_started`: no meaningful implementation yet.
- `stubbed`: route or shell exists, but behavior is placeholder-level.
- `fixture_partial`: behavior is backed by fixtures or in-memory repositories.
- `api_contract_ready`: response/request shape is locked by tests.
- `parity_ready`: fixture parity tooling exists and passes against AI-CRM Next TestClient.
- `postgresql_ready`: SQLAlchemy repository or Alembic schema exists, but production PostgreSQL is not connected.
- `integration_test_ready`: explicit integration test framework exists and is opt-in.
- `production_ready`: real production data, real external dependencies, real deployment validation, and rollback are proven.

No current module is `production_ready`.
