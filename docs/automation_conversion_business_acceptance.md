# 自动化转化业务验收说明

## 一句话目标

把“加好友 -> 提交问卷 -> 开通试用 -> 激活/未激活分池 -> 人工改判 -> OpenClaw 介入 -> 人工确认成交退出营销”收口成一套可验收的自动化转化主线。

## 6 个池子定义

1. 新用户池
   渠道用户刚进入 CRM，还没有提交问卷，或已提交问卷但还没有明确的试用开通事实。
2. 未激活普通池
   已提交问卷、已有试用开通事实、未激活、当前跟进类型是普通跟进。
3. 未激活重点跟进池
   已提交问卷、已有试用开通事实、未激活、当前跟进类型是重点跟进。
4. 激活普通池
   已激活、当前跟进类型是普通跟进。
5. 激活重点跟进池
   已激活、当前跟进类型是重点跟进。
6. 沉默池
   在前 5 个池子中的任一池停留超过配置天数后自动进入；当前版本只做留存记录，不做主动经营。

## 核心业务规则

- 问卷只负责第一次分流，只输出普通跟进或重点跟进。
- 人工改判永远优先于问卷初判。
- 同一用户同一时间只能在一个当前池子里。
- 切池后，上一池动作立刻停止，后续动作一律按新池子走。
- 人工确认成交后直接退出全部营销。

## 普通路径

1. 客户进入 CRM，当前状态为新用户池。
2. 客户提交问卷，问卷里必须填写手机号。
3. 问卷命中普通跟进后，系统保留普通跟进初判结果。
4. 一旦存在试用开通事实，客户进入未激活普通池。
5. 外部系统回写激活后，客户自动进入激活普通池。
6. 运营可在侧边栏把客户临时切到重点跟进；再次切回普通跟进后，当前池子同步调整。
7. 运营人工确认成交后，客户退出全部营销。

## 重点跟进路径

1. 客户进入 CRM，当前状态为新用户池。
2. 客户提交问卷并填写手机号。
3. 问卷命中重点跟进后，系统保留重点跟进初判结果。
4. 一旦存在试用开通事实，客户进入未激活重点跟进池。
5. OpenClaw 可以直接按重点跟进池发起 CRM 群发。
6. 重点跟进池客户来新消息时，CRM 自动把客户档案、问卷摘要、标签、最近聊天和当前阶段推送给 OpenClaw webhook。
7. 外部系统回写激活后，客户自动进入激活重点跟进池。
8. 运营人工确认成交后，客户退出全部营销。

## 沉默池定位

- 沉默池只承接超时未出池的客户。
- 沉默池当前只做留存记录。
- 沉默池默认不开放群发。
- 沉默池当前不对接 OpenClaw 主动经营动作。

## OpenClaw 的作用边界

- OpenClaw 不再先读池子名单再自己发消息。
- OpenClaw 通过 MCP 工具直接调用 CRM 的按池子群发能力。
- CRM 负责筛池、调用现有私聊发送链路、记录发送记录。
- 对重点跟进池客户，CRM 在收到客户新消息后自动推送 webhook 给 OpenClaw。
- CRM 只负责推送上下文，不替 OpenClaw 做复杂判断。

## 人工的作用边界

- 人工可以在侧边栏把普通跟进和重点跟进互相切换。
- 人工改判优先于问卷初判。
- 人工确认成交是退出营销的唯一成交动作。
- 人工不需要直接维护试用开通事实和激活事实，试用开通由明确事实源承载，激活由外部 webhook 回写。

## v1 限制说明

- 当前池子群发和重点消息 webhook 已支持多个负责人，并按真实 `owner_userid` 生效。
- 池子群发当前已支持纯文本、纯图片、纯附件、文本 + 图片、文本 + 附件、图片 + 附件、文本 + 图片 + 附件。
- 附件当前沿用现有私聊发送底层结构，本轮正式开放的是 `file` 类型 attachment 对象。
- webhook 当前没有复杂重试队列 / 死信系统。
- 激活回写当前只按手机号匹配客户。
- `trial_opened` 事实源当前复用现有 `user_ops_pool_current.current_status='lead_trial'`。
- 代码里仍保留少量内部兼容命名，例如 `signup_conversion_v1`，但不对业务页面暴露。
