# 自动化转化 Open Issues

## 当前已知但不阻塞业务验收的问题

1. 池子群发当前已支持文本、图片和附件，但仍未提供素材库/富文本能力
   - 当前 MCP 已支持纯文本、纯图片、纯 `file` 附件、文本 + 图片、文本 + 附件、图片 + 附件、文本 + 图片 + 附件；但仍不提供素材库、富文本编辑器和复杂模板编排，也未正式开放其他 attachment 类型。

2. webhook 当前没有复杂重试队列 / 死信系统
   - 当前已经支持 delivery log、状态机、手动重试、到期重试和 jobs 面板，但还没有更复杂的异步队列与死信补偿。

3. 激活回写当前只按手机号
   - v1 只支持手机号匹配客户，不支持 external_userid / unionid 双写入口。

4. 沉默池当前只记录不经营
   - 沉默池当前不开放群发，也没有专门经营动作。

5. 仍存在少量内部兼容命名
   - 例如 `signup_conversion_v1`、`top_threshold`、旧兼容 API 别名；当前不影响业务页面和验收链路。

6. `trial_opened` 事实源是基于现有表的最小落地方案
   - 当前复用 `user_ops_pool_current.current_status='lead_trial'`，这是明确事实源，但还不是更完整的独立事件体系。
