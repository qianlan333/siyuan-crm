# Issue #108 RAUTH 简化替换实施计划

## 决策边界

- 部署模型：single-tenant 私有化部署，不新增独立授权服务或 runtime unit。
- 人员身份：企微 OAuth/网页登录是唯一上游，继续使用服务端 Session、RBAC、
  `session_version`、CSRF 和必要的 route-bound action token。
- Agent/API Client：每个调用方独立 `client_id/client_secret`，数据库只存强哈希；
  通过 TLS `client_credentials` 换取默认 30 分钟的签名 JWT，不签发 refresh token。
- Worker/Timer：同进程直接传入受控 service `AuthContext`；独立进程和 CLI 使用
  workload 级 client credentials，不再保存多套共享 Bearer。
- AI-CRM 自有 Webhook：HMAC-SHA256 绑定 timestamp、event_id/nonce 和 raw body，
  强制时间窗与持久化防重放；供应商官方 callback/OAuth 协议保持不变。
- 回滚：恢复上一 verified release、对应配置和数据库备份，不保留旧新双栈或 fallback。

## 明确删除

- 独立 Authorization Server、OIDC discovery/consent/PKCE。
- opaque token、online introspection、refresh-token family、Token Exchange。
- DPoP、mTLS、FAPI 和多租户/开放平台能力。
- query/path token、共享万能 Token、route-local credential validator、旧 fallback。
- MCP、archive、group broadcast、callback、automation worker 等静态 Bearer 环境变量。

## 单分支、单 PR、单 release 顺序

1. 固化精确 credential inventory，并将供应商凭据单列 allowlist。
2. 建立轻量 `auth_api_clients`、一次性 secret 展示、强哈希、scopes、allowed CIDR、
   `auth_version`、enabled 与轮换迁移。
3. 建立短期 JWT 签发/本地验签和 client 状态缓存；校验 `iss/aud/sub/client_id/scope/
   iat/exp/jti/auth_version`。
4. 统一人员 Session、API client、service、Sidebar/Action Grant、Webhook 到一个
   `AuthContext` 和 route middleware；业务 Handler 不再读取凭据。
5. 一次性迁移所有 Agent、Worker、Timer、CLI、前端和文档，并删除旧代码与配置。
6. 建立静态门禁：业务层 credential 读取为零、旧 env/query/path/fallback 引用为零。
7. 运行 PostgreSQL migration/rollback、JWT/轮换/停用/越权、Webhook 篡改/过期/重放、
   Campaign Agent 权限、供应商回归、全量 CI。
8. bootstrap 所有生产 client，发布一个 exact SHA，重启 runtime，执行 count-only
   reconciliation；失败整包回滚。

## Campaign Agent 最小权限

允许：`customer.resolve.read`、`customer.read.limited`、`material.read`、
`material.create`、`campaign.draft.create`、`campaign.status.read`。

约束：当前 corp、owner allowlist、recipient 上限、
`send_path=ai_assist_pending_review`，结果只能是 draft/pending_review，真实 broadcast
job 必须为 0。

禁止：approve、start、broadcast.execute、admin.global、退款、密钥管理和 PII 导出。

## 验收证据

- 人员 Session 登录、CSRF、角色能力、停用/降权和 `session_version` 即时吊销。
- client secret 正误、轮换、停用、旧 `auth_version`、过期、issuer/audience/scope 越权。
- Worker workload 交叉 Token 拒绝；同进程任务不经 HTTP 给自己发 Token。
- Webhook body 篡改、过期 timestamp、重复 event_id/nonce 全部拒绝。
- Campaign Agent 可创建 draft，但审批/启动/直接发送均为 403。
- secret/token 不明文进入普通配置、URL、日志、审计或数据库。
- 全量 CI、migration、exact-SHA deploy、readiness 和整包 rollback 证据完整。
