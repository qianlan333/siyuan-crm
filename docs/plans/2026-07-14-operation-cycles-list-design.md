# 运营闭环一级页极简列表设计

## 目标

一级页只回答三个问题：有哪些运营任务、每项任务本周进行到哪一步、从哪里进入明细。删除原有发布会式主视觉、核心漏斗、结论与技术数据源，避免在列表页重复二级页的信息。

页面沿用 Admin Shell 的 SaaS 管理列表模式，表格固定为「任务名称 / 本周进度 / 操作」三列。任务由 Agent 在外部创建，因此本页不提供新建入口；“暂停”和“删除”仅作为禁用占位，“明细数据”链接到现有策略详情页，不新增写接口或外部动作。

## 本周进度口径

服务端按策略时区计算当前自然周，周一为起点、周日为终点。只有最近一次运行时间落在本周时才参与本周进度；否则四个阶段全部回到未开始。每次打开页面都会重新计算，因此跨周后无需修改存量快照。

进度固定为四段：任务准备、发送执行、结果复盘、优化确认。阶段只使用已上报的执行、人审、发送与优化状态推导，不补造业务结果。完成段为绿色，当前段为蓝色，未开始段为灰色；异常或尚未完成仍停留在对应当前段，不被错误标记为完成。列表只展示居中的四段流程条，不额外展示进度结论或周日期。

## 边界、验证与回滚

Capability owner 保持 `aicrm_next.operation_cycles`，仅修改 `/admin/operation-cycles` 的展示模型、模板与命名空间样式。API、数据库、上报契约、二级策略页、三级运行页、权限和路由所有权均不改变，也不涉及真实外部调用或生产配置。

测试覆盖本周运行、跨周自动归零、禁用操作按钮、明细链接、空列表与响应式样式。回滚只需回退本次模板、展示辅助函数、样式和测试提交，不需要数据回滚。
