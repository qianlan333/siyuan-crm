# 工作流 C — 聊天上下文推荐

适用场景：用户在和客户聊天时（或回顾对话时），希望从素材库找几张合适的图发出去。这是图片素材库 AI 化的**核心目标场景**。

## 输入形态

用户可能给：

- **完整对话片段**："客户 A 的最近 3 条消息：...，请推荐图片"
- **场景描述**："客户在表达对效果的质疑"
- **明确意图**："找 3 张体现信任建立的好评截图"

## 步骤

### 1. 提取语义信号

从对话 / 场景描述里抽出：

- **客户痛点 / 情绪**：质疑 / 犹豫 / 信任不足 / 价格敏感 / 担心效果 / 急迫 / 已认同
- **决策阶段**：认知（不知道产品）/ 兴趣（在了解）/ 评估（在比较）/ 行动（要下单）/ 复购（已成交）
- **需要的回应类型**：社会证明（好评）/ 数据证明（对比图）/ 情感共鸣（同类案例）/ 紧迫性（限时活动）/ 操作指引（怎么用）

把这些信号转成查询关键词。

### 2. 拉候选元数据

```
image_library_list({
  tags: ["信任建立", "好评"],          // 1-3 个
  category: "好评截图",                  // 可选；用户明确要哪类才传
  q: "宝妈",                              // 可选；命中 description / name 的关键词
  limit: 10
})
```

**注意**：`tags` 是 OR 语义（命中任一），所以传太多反而稀释。3 个以内够。

如果第一次返回 `count < 3`，放宽条件再查一次（去掉 category，或减 tags）。

### 3. 元数据重排

返回的每张图有 `description` + `tags` + `category` + `ai_metadata`。**不读 base64**，光看元数据就能判断相关度。

打分思路（不严格，凭语义）：

- description 直接说"适合在 X 场景使用" → 高分
- tags 命中用户给的痛点维度多个 → 高分
- ai_metadata.vision_confidence 高 → 加分
- 全空（极少见）→ 0 分跳过

输出 top 3-5 张，每张给一个**一句话引用理由**：

> #234 - 微信聊天截图，宝妈用户表达对效果的肯定。tags: 好评/信任建立/宝妈群体。
> **推荐理由**：直接呼应客户对效果的质疑，目标群体匹配。

### 4.（可选）vision 二次确认

如果 top 1-2 是关键决策（比如要发给重点客户），**只对这两张**调：

```
image_library_get({image_id, with_data: true})
```

让 vision 再看一眼图，确认 description 没失真。

但**不要**对 top 5 全做这一步 —— vision 调用是最贵的环节，应该只在确认环节用。

### 5. 输出推荐结果

```markdown
**推荐 3 张图（按相关度排序）**：

1. **#234** 「宝妈好评截图」
   - description: 微信聊天截图，宝妈用户晒孩子使用产品 30 天后的进步...
   - tags: 好评 / 信任建立 / 宝妈群体
   - 推荐理由：直接呼应客户对效果的质疑，目标群体匹配
   - 引用方式：image_library_id=234（campaign / 客服回复都能用）

2. **#198** 「数据对比图」
   - ...

3. **#155** 「同类案例」
   - ...

**未命中候选的备注**：今天没有「价格说服」类的图，如果客户后续聊到价格，可能要新上传一张。
```

### 6. 记录 / 反馈循环（未来）

短期不做。未来 CRM 那边可以加 endpoint 让用户标记"这张图实际发了 / 没发 / 客户回应正面"，回流到 ai_metadata 的 `effectiveness_log` 字段，让 Skill 下一次推荐更准。

## 与 list 的关系

`image_library_list` 本身就支持 q / tags / category 过滤；**本工作流没有引入新的 search endpoint**，是把 list 当推荐底座，加上 Skill 侧的语义重排。

未来如果想做 embedding 召回，可以在 CRM 加一个 `image_library_recommend` 工具，但目前 list + tag 检索 + Skill 重排足够覆盖侧边栏推荐场景。

## 反模式（避免）

- ❌ **直接给 base64 vision 选图** —— 太贵，先用元数据筛
- ❌ **不限 limit，拉一大批** —— 重排时 token 撑爆
- ❌ **只选 1 张** —— 客户不一定要发图，给候选让用户自己挑
- ❌ **强行造推荐** —— 如果 list 真的返回空，老实告诉用户"素材库里没合适的"，不要硬挑一张不相关的
