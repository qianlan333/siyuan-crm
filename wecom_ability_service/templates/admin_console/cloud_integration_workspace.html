Claude Code · 接入凭证

把下面的连接信息粘到你的 Claude Code MCP 配置里，重启就能用 AI 直接驱动 CRM。

① MCP 端点

https://<your-crm-host>/mcp

② Bearer Token

在 CRM 的 app_settings.MCP_BEARER_TOKEN 里配置（运维负责）。请从工程师那里获取。

Authorization: Bearer <TOKEN>

③ Claude Code 配置示例

编辑你的 Claude Code MCP 配置文件（~/.config/claude-code/mcp.json 或当前项目的 .mcp.json）：

{
  "mcpServers": {
    "crm": {
      "type": "http",
      "url": "https://<your-crm-host>/mcp",
      "headers": {
        "Authorization": "Bearer <TOKEN>"
      }
    }
  }
}

④ 喂给 Claude Code 的方法论文档（重要）

把下面 4 个文档整段粘到你项目的 CLAUDE.md，让 Claude Code 一开机就知道怎么用 CRM：

  • docs/claude_code_integration/README.md — 接入总览 + 立即可用的 5 句话场景
  • docs/claude_code_integration/tools.md — 21 个工具的标准用法
  • docs/claude_code_integration/patterns.md — 9 个典型运营场景方法论
  • docs/claude_code_integration/rules.md — 8 条必守规则（防骚扰 / 互斥 / token / 人工 review）
  • docs/claude_code_integration/troubleshooting.md — 排错速查

⑤ 试一下

在 Claude Code 里输入：

列一下当前所有等待审阅的 Campaign，告诉我每个的目标和候选人数。

Claude Code 应当：调 list_campaigns → 调 get_campaign 看每条详情 → 用业务语言回报你。

三端协作图

                 ┌──────────────────────┐
                 │  Claude Code（你）   │  ← 提目标、写策略、调工具
                 └──────────┬───────────┘
                            │ MCP HTTP
                ┌───────────▼────────────┐
                │  CRM（能力服务方）      │  ← 池子、画像、互动、调度、发送
                └─────────┬──────────────┘
                          │ 工单
                ┌─────────▼───────────────┐
                │  话术 AI（DeepSeek）    │  ← 写多变体话术
                └─────────────────────────┘

必读三件事

  1. 真发只能在 CRM 后台。Claude Code 永远只能 propose，最后那一下"启动"必须在 /admin/cloud-orchestrator/campaigns 页人工点击。
  2. 互斥保障是数据库 UNIQUE 约束。同一个用户哪怕被 N 个分层命中，在同一个 Campaign 内只占 1 条节奏。Claude Code 给每个 segment 设 priority 决定谁优先抢人。
  3. 每次操作带 trace_id。出问题在 可观察性页 输 trace_id 一查到底。
